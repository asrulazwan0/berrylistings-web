# BerryListings — Web Frontend

React + TypeScript frontend for the BerryListings property management platform. Design reference in `../berrylistings-ui/`.

## Stack
- React 19
- TypeScript
- Vite (build tool)
- Consumes `berrylistings` REST API (Express backend)

## Commands
```bash
npm run dev         # Vite dev server
npm run build       # Production build
npm run preview     # Preview production build
npm run lint        # ESLint
npm run format      # Prettier
npm test            # Test suite
```

## Conventions
- All API calls target the berrylistings backend (`/api/v1/...`)
- Auth: Google OAuth → backend JWT (bearer token in Authorization header)
- Admin routes require authenticated admin user
- Use design tokens from `berrylistings-ui/css/tokens.css` as visual reference
- Component files: `PascalCase.tsx` in `src/components/`
- Pages: `src/pages/` — one per route
- API layer: `src/api/` — typed fetch wrappers

## Reference
- Design: `../berrylistings-ui/` — HTML/CSS reference implementation
- Backend: `../berrylistings/` — API with Swagger docs at `/docs`
- Property types: HOUSE, CONDO, TOWNHOME, LAND
- Property statuses: DRAFT, ACTIVE, PENDING, SOLD
